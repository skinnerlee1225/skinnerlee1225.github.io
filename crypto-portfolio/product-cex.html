product overview

產品定位與設計挑戰

這是一個 白標 (White Label) 中心化交易所 — 一套產品同時服務多個 B 端客戶 (品牌方)，每個客戶可以快速套用、客製化品牌、調整費率、設定地區合規。我們不是 Binance 那種百人+ 工程團隊的規模（團隊約 30 人），但要做到主流 CEX 的功能完整度（現貨 / 合約 / 理財）。設計挑戰是：在資源有限的前提下，每個客戶都要當作獨立交易所運作，但底層共用一套撮合引擎、帳本系統、風控引擎。

挑戰 01多租戶資料隔離

每個 B 端客戶的用戶資料、交易紀錄、資金帳本必須完全隔離，但又共用同一套撮合引擎與冷熱錢包架構。最終採用 tenant_id 分庫策略 — 業務邏輯層共用，資料層按租戶 sharding。

挑戰 02撮合引擎的效能極限

單交易對需達 10 萬 TPS、延遲 < 5ms。最終採用記憶體撮合 (Go/Rust 實作) + Kafka 落地 + ClickHouse 行情聚合。前期用 Go 寫第一版，瓶頸時用 Rust 改寫核心。

挑戰 03合約強平與保險基金

125x 槓桿下，強平機制必須既快又準。設計三層強平：(1) 市價減倉 → (2) 保險基金補足 → (3) ADL 自動減倉。標記價格取三所加權平均，防止單一交易所插針造成不合理強平。

關鍵決策 #1
Go vs Rust 撮合引擎

第一版選 Go — 團隊熟、開發快、可達 5 萬 TPS。預留 Rust 重寫 hook 點，當壓測瓶頸時可平滑遷移。

關鍵決策 #2
K 線自建 vs TradingView

整合 TradingView Charting Library — 省 3~4 週開發、用戶熟悉度高、技術指標完整。授權成本可接受。

關鍵決策 #3
KYC 自建 vs 第三方

接 SumSub / Jumio 第三方服務 — 省 4+ 週開發、合規覆蓋更廣、人臉辨識精度更高。

關鍵決策 #4
Web 優先 vs Web+App 並行

MVP 階段只做 Web — 省 4+ 週、Web 已覆蓋 70% 交易需求。App 留到 Phase 2 用 Flutter 一次出包雙平台。

system architecture

9 大系統模組

每個模組獨立部署、透過內部 API 與 Kafka 消息隊列互通。下方為各模組職責與技術選型。

MODULE 01
用戶系統 User Service

註冊、登入、2FA、KYC 三級、設備管理、API Key。

MODULE 02
資產系統 Asset Service

充值、提現、餘額管理、三帳戶劃轉、資產快照。

MODULE 03
現貨交易 Spot Trading

記憶體撮合引擎，10 萬 TPS，限價/市價/止盈止損/OCO。

MODULE 04
合約交易 Futures

USDT 本位 + 幣本位永續，1x~125x 槓桿，逐倉/全倉。

MODULE 05
理財系統 Earn

活期 / 定期 / Staking / 雙幣投資，APY 計息。

MODULE 06
風控系統 Risk

KYC / AML / 規則引擎 + ML 模型雙軌並行。

MODULE 07
手續費 Fee Service

VIP 分級、Maker/Taker、平台幣抵扣 25% 折扣。

MODULE 08
通知系統 Notification

Email / Push / SMS / 站內信，事件驅動推送。

MODULE 09
後台管理 Admin

用戶管理、資產監控、交易監控、操作日誌雙人審批。

core features

核心功能設計

2.1資產頁 Wallet

頂部「總資產估值」默認 USD 計價、支援 BTC/CNY/EUR 切換；估值基於即時中間價 (Bid+Ask)/2。提供「隱藏小額資產」開關（< $1 自動隱藏）+ 眼睛圖示一鍵隱藏餘額。

三本帳戶分區展示

帳戶類型顯示內容操作入口
現貨帳戶各幣種可用 + 凍結餘額交易 / 充值 / 提現 / 劃轉
合約帳戶總保證金 / 可用保證金 / 未實現盈虧劃轉 / 交易
理財帳戶活期 + 定期 + 累計收益申購 / 贖回 / 劃轉

2.2現貨交易 Spot Trading

整合 TradingView 圖表庫，支援 1m/5m/15m/1h/4h/1D/1W。內建 MA/EMA/BOLL/MACD/RSI/VOL 等技術指標。Orderbook WebSocket 推送，延遲 < 50ms，顯示最佳 20 檔買賣價量 + 累計深度條形圖。

四種訂單類型

訂單類型說明參數
限價單 Limit指定價格掛單等待成交價格 + 數量
市價單 Market以最優價即時成交數量（或金額）
止盈止損 Stop-Limit觸發價到達後掛限價單觸發價 + 限價 + 數量
OCO限價 + 止損二選一，一方成交另一方自動取消限價 + 止損觸發 + 止損 + 數量

2.3合約交易 Futures Trading

USDT 本位 + 幣本位永續合約。槓桿滑桿 1x~125x，BTC/USDT 最高 125x、山寨幣最高 20~75x。逐倉/全倉模式可在無持倉時切換。

強平與標記價格

持倉面板常駐顯示強平價格 (紅色)。標記價格接近強平價時 (差距 < 5%)，觸發 Push 通知 + 頁面閃爍警告。標記價格 = 三所 (Binance/OKX/Bybit) 加權平均，防插針。

強平價格（逐倉做多）= 開倉價 × (1 - 1/槓桿 + MMR)
例：BTC $50,000 開多，10x 槓桿，MMR = 0.5%
強平價 = 50,000 × (1 - 0.1 + 0.005) = $45,250

2.4理財 Earn

活期：T+0 贖回、利率浮動、現貨閒置可自動申購。定期：7/14/30/60/90/120 天，到期自動到帳，可勾選自動續期。Staking：PoS 鏈原生質押 (ETH/SOL/ADA/DOT)，鎖倉期間每日發放收益。

日收益 = 持倉金額 × APY / 365
APR vs APY 轉換: APY = (1 + APR/n)^n - 1  （n = 複利頻率）

risk & security

風控與安全設計

6.1KYC 三級分流

等級要求權限
Lv0 (未驗證)僅註冊瀏覽行情，無法交易/充提
Lv1 (基礎)身份證 + 人臉辨識充提 ≤ 2 BTC/日，合約 ≤ 20x
Lv2 (進階)地址證明 + 影片驗證充提 ≤ 100 BTC/日，合約 ≤ 125x
機構帳戶營業執照 + 法人 + 受益人自定義額度

6.2冷熱錢包架構

熱錢包僅保留平台總資產 2%~5% (滿足日常提現)。冷錢包保管 95%~98%。冷錢包私鑰分片存於不同地理位置 (Shamir's Secret Sharing / MPC 多簽)。每日自動對帳 (鏈上 vs 帳本)，差異即時告警。

6.3合約風險限額分級

倉位越大，最大可用槓桿越低（階梯式限額），防止巨鯨單方向操縱市場。

6.4提現風控三層防線

第一層 用戶端：2FA 雙重驗證、提現白名單、防釣魚碼。

第二層 系統端：新地址 24h 鎖定、累計提現限額 (按 KYC 等級)、異常行為偵測。

第三層 人工端：大額 (> $10K) 人工審核 (2h SLA)、可疑交易人工複查、凍結帳號需雙人審批。
